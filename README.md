# 🎨 Visual Workspace Platform\n\nA modern, collaborative web platform combining mind mapping, flowcharting, project architecture planning, and structured documentation into one seamless experience.\n\n**Inspired by:** Miro, Notion, Whimsical, FigJam\n\n## ✨ Key Features\n\n### 🎯 Core Features\n- **Infinite Collaborative Canvas** - Smooth, zoomable, pannable workspace\n- **Visual Block System** - Drag-and-drop nodes with rich content support\n- **Smart Connections** - Create directional flows between blocks\n- **Nested Architecture** - Create hierarchical structures with sub-blocks\n- **Real-time Collaboration** - Multiple users editing simultaneously\n- **Live Cursors & Presence** - See who's editing what in real-time\n\n### 📋 Block Features\n- Rich text titles and descriptions\n- Expandable notes sections\n- Checklists with progress tracking\n- File attachments\n- Custom tags and labels\n- Color customization\n- Emoji/icon support\n- Priority levels (Low, Medium, High, Critical)\n- Status tracking (To-do, In Progress, Review, Done)\n- Deadline management\n- @Mentions and threaded comments\n\n### 👥 Collaboration Features\n- Real-time multi-user editing\n- Permission-based access (Owner, Admin, Editor, Viewer)\n- Email/link-based sharing\n- Team member management\n- Activity timeline and audit logs\n- Version history with restore capability\n- Comment threads on blocks\n\n### 🎛️ Advanced Features\n- **Search** - Global search across all blocks and notes\n- **AI-Assisted** - Smart node generation and suggestions\n- **Auto-Layout** - Intelligent block arrangement\n- **Templates** - Pre-built templates for workflows\n- **Keyboard Shortcuts** - Power user shortcuts\n- **Dark/Light Themes** - Complete theme support\n- **Export** - PNG, PDF, Markdown, JSON formats\n- **Offline Mode** - Work offline with auto-sync\n- **Notifications** - Real-time alerts and mentions\n- **Focus Mode** - Isolate and zoom into branches\n- **Presentation Mode** - Present your workspace\n\n## 🏗️ Architecture\n\n### Tech Stack\n\n**Backend:**\n- Node.js 18+ with Express.js\n- MongoDB for data storage\n- Socket.io for real-time communication\n- Redis for caching\n- JWT for authentication\n\n**Frontend:**\n- React 18+ with TypeScript\n- Redux Toolkit for state management\n- Custom Canvas API for rendering\n- Tailwind CSS for styling\n- Framer Motion for animations\n\n**DevOps:**\n- Docker & Docker Compose\n- MongoDB 6+\n- Redis 7+\n- Nginx for reverse proxy\n\n## 📁 Project Structure\n\n```\nvisual-workspace-platform/\n├── backend/                    # Express.js server\n│   ├── server.js              # Main server file\n│   ├── models/                # Database schemas\n│   ├── routes/                # API endpoints\n│   ├── controllers/           # Business logic\n│   ├── middleware/            # Express middleware\n│   ├── websocket/             # Socket.io handlers\n│   └── package.json\n│\n├── frontend/                   # React application\n│   ├── src/\n│   │   ├── components/        # React components\n│   │   ├── pages/            # Page │   ├── styles/           # CSS files\n│   │   └── App.jsx           # Main component\n│   ├── vite.config.js\n│   └── package.json\n│\n├── shared/                     # Shared utilities\n│   ├── clone https://github.com/yourusername/visual-workspace-platform.git\ncd visual-workspace-platform\n```\n\n2. **Backend Setup**\n```bash\ncd backend\nnpm install\ncp .env.example .env\n# Edit .env with your configuration\nnpm run dev\n```\n\n3. **Frontend Setup**\n```bash\ncd frontend\nnpm install\ncp .env.example .env\nnpm run dev\n```\n\n4. **Access the Application**\n```\nhttp://localhost:3000\n```\n\n## 🐳 Docker Deployment\n\n### Using Docker Compose\n\n```bash\n# Start all services\ndocker-compose up -d\n\n# Stop all services\ndocker-compose down\n\n# View logs\ndocker-compose logs -f\n```\n\nAccess:\n- Frontend: http://localhost:3000\n- Backend: http://localhost:5000\n- MongoDB: localhost:27017\n\n## 📚 API Endpoints\n\n### Authentication\n- `POST /api/auth/signup` - Register new user\n- `POST /api/auth/login` - Login user\n- `GET /api/auth/me` - Get current user\n\n### Projects\n- `GET /api/projects` - Get all user's projects\n- `POST /api/projects` - Create new project\n- `GET /api/projects/:id` - Get project details\n- `PATCH /api/projects/:id` - Update project\n- `DELETE /api/projects/:id` - Delete project\n\n### Blocks\n- `POST /api/blocks` - Create new block\n- `GET /api/blocks/:id` - Get block details\n- `PATCH /api/blocks/:id` - Update block\n- `DELETE /api/blocks/:id` - Delete block\n\n### Connections\n- `POST /api/connections` - Create connection\n- `DELETE /api/connections/:id` - Delete connection\n\n### Activity & Collaboration\n- `GET /api/projects/:id/activity` - Get activity log\n- `GET /api/projects/:id/versions` - Get version history\n- `POST /api/projects/:id/versions` - Create version snapshot\n\n### Search\n- `GET /api/projects/:id/search?query=` - Search blocks\n\n## 🔧 Environment Variables\n\n### Backend (.env)\n```\nNODE_ENV=development\nPORT=5000\nMONGO_URI=mongodb://localhost:27017/visual-workspace\nJWT_SECRET=your-secret-key\nJWT_EXPIRE=7d\nREDIS_URL=redis://localhost:6379\nSOCKET_IO_ORIGIN=http://localhost:3000\n```\n\n### Frontend (.env)\n```\nVITE_API_URL=http://localhost:5000\nVITE_SOCKET_URL=http://localhost:5000\nVITE_ENV=development\n```\n\n## 🎮 Keyboard Shortcuts\n\n| Shortcut | Action |\n|----------|--------|\n| `Ctrl+N` | Create new block |\n| `Delete` | Delete selected block |\n| `Ctrl+Z` | Undo |\n| `Ctrl+Shift+Z` | Redo |\n| `Ctrl+F` | Search |\n| `Ctrl+/` | Show shortcuts |\n| `Ctrl+1` | Zoom to 100% password: String (hashed),\n  name: String,\n  avatar: String,\n  preferences: Object,\n  createdAt: Date,\n  updatedAt: Date\n}\n```\n\n### Project\n```javascript\n{\n  _id: ObjectId,\n  name: String,\n  description: String,\n  ownerId: ObjectId,\n  teamMembers: Array,\n  blocks: Array<ObjectId>,\n  connections: Array<ObjectId>,\n  visibility: String, // 'private', 'team', 'public'\n  thumbnail: String,\n  shareToken: String,\n  activityAt: Date,\n  createdAt: Date,\n  updatedAt: Date\n}\n```\n\n### Block\n```javascript\n{\n  _id: ObjectId,\n  projectId: ObjectId,\n  parentId: ObjectId,\n  title: String,\n  description: String,\n  content: String,\n  notes: String,\n  checklists: Array,\n  attachments: Array,\n  tags: Array<String>,\n  color: String,\n  icon: String,\n  priority: String,\n  status: String,\n  deadline: Date,\n  position: { x: Number, y: Number },\n  size: { width: Number, height: Number },\n  children: Array<ObjectId>,\n  createdBy: ObjectId,\n  createdAt: Date,\n  updatedAt: Date\n}\n```\n\n## 🧪 Testing\n\n### Running Tests\n```bash\n# Backend\ncd backend\nnpm test\n\n# Frontend\ncd frontend\nnpm test\n```\n\n### Test Coverage\n- Unit tests
